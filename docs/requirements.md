# Requirements

## 1. 프로젝트 개요

**프로젝트명:** Orderly  
**목표:** Nest.js 백엔드 아키텍처 학습과 이직 포트폴리오 구축  
프론트엔드는 **Next.js + TypeScript**로 최소한의 기능만 제공하고, 백엔드에서는 **Nest.js + TypeScript** 기반의 MSA와 EDA 아키텍처 실습에 집중한다.  
설계 전반은 **DDD(Domain Driven Design)** 원칙을 적용해 바운디드 컨텍스트 단위로 기능을 확장한다.
설계 전반은 **DDD(Domain Driven Design)** 원칙을 따르며 도메인/바운디드 컨텍스트 단위로 기능을 확장한다.

---

## 2. 핵심 기능

### [Backend - Nest.js]

- **MVP 1 (주문 조회 중심)**
  - **회원 관리**: 가입, 로그인, JWT 인증
  - **주문 처리**: 주문 생성, 상태 변경, 이벤트 발행
  - **주문 조회 API**: CQRS 기반 읽기 모델 제공
- **MVP 2 (결제 및 카탈로그 확장)**
  - **상품 카탈로그**: 상품 등록, 수정, 조회
- **결제 모듈**: Toss Payments 기반 결제 및 환불 처리
  - **이벤트 브로커 연동**: 주문/결제 도메인 간 이벤트 흐름 정립
- **MVP 3 (고객 경험 강화)**
  - **리뷰 및 피드백**: 구매 후 사용자 리뷰 작성
  - **관리자 기능**: 주문/상품 통합 관리
  - **관찰 가능성**: 로그 및 트레이싱 기반 모니터링 도입

### [Frontend - Next.js]

- **필수 화면만 구현**: 주문 내역 조회 및 기본 인증 UI
- **백엔드 검증용 최소 UI**: 주문 생성, 상태 확인 플로우 중심
- **고급 UI 기능은 후순위**: 장바구니, 관리자 전용 화면은 추후 확장

## 3. 기술 요구사항

- **모노레포 관리:** Turborepo
- **패키지 매니저:** npm
- **개발 환경:** docker-compose 기반 로컬 컨테이너 오케스트레이션
- **DB:** PostgreSQL (서비스별 개별 인스턴스, TypeORM 기반)
- **이벤트 브로커:** Apache Kafka
- **CI/CD:** GitHub Actions
- **테스트:** Jest + Testing Library
- **API 문서화:** Swagger(OpenAPI 3.0)
- **코드 품질:** ESLint + Prettier + Husky (pre-commit hook)

## 4. 사용자 유형

| 역할       | 설명                                 |
| ---------- | ------------------------------------ |
| **고객**   | 상품을 조회, 주문, 결제, 리뷰 작성   |
| **관리자** | 상품 및 주문을 관리, 고객 요청 처리  |
| **시스템** | 자동 결제 처리 및 주문 상태 업데이트 |

---

## 5. 비기능 요구사항

- 모든 API는 REST 규약 준수
- 주요 로직 단위 테스트 커버리지 80% 이상
- 서버 응답 시간 200ms 이하 유지
- ESLint 규칙 준수 및 자동 포맷팅
- 에러는 공통 에러 핸들러로 관리
- **MSA/EDA 적용**: 서비스 간 경계를 명확히 하고 이벤트 기반 통신 패턴 유지
